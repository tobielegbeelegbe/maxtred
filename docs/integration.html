Integrating with existing project

We recommend using MatX for starting a new project. But if you want to use it with your existing project, follow below steps.

1. Merge package.json: Merge your project's package.json with Matx's package.json. You can merge manually by copying the dependencies from Matx to your project's package.json. Or you can use merge-package.json

2. Copy files & folders: Make sure your app has src and src/app folder. Copy following folders and files to your project.

  • src/app/auth
  • src/app/components
  • src/app/context
  • src/app/hooks
  • src/app/views/session
  • src/app/navigation.js
  • src/app/routes.jsx
  • src/app/utils
  • src/config.js
  • src/history.js

3. App.js Copy content from Matx's src/app/App.js to your project's src/app/App.js, You can keep the necessary code requred by your project.

4. Define Child Routes: Matx has all the pages inside views folder. It doesn't matter where you keep your pages. But you need to maintain route definition structure.

For example, you have order related pages in order folder. You should have OrderRoutes.js for those pages.

  • OrderList.jsx
  • SingleOrder.jsx
  • OrderRoutes.js

5. Define Root Route: MatX has a main route file app/routes.jsx that you've already copied to your project. Now you have to import all the child routes(e.x. OrderRoutes.js) into app/routes.jsx.

5. Import Material Icons: Import material icons in your index.html
